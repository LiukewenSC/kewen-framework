# 权限控制系统

## 概述

Kewen Framework 的权限控制系统是一个基于 RBAC（Role-Based Access Control）模型的完整权限管理解决方案。通过注解驱动的方式，实现了菜单权限和数据权限的精细控制。

## 核心特性

### 1. 注解驱动权限控制

通过简单的注解即可实现权限验证，无需编写复杂的权限校验代码：

- `@AuthMenu`：菜单权限控制
- `@AuthDataRange`：数据范围权限控制
- `@AuthDataOperation`：数据操作权限控制
- `@AuthDataAuthEdit`：数据权限编辑

### 2. RBAC 权限模型

实现了标准的用户-角色-部门权限体系：

- **用户（User）**：系统使用者
- **角色（Role）**：权限的集合
- **部门（Dept）**：组织架构
- **菜单（Menu）**：路由权限和 API 权限

### 3. Spring Security 集成

- JSON 格式登录
- Token 认证
- Session 管理
- 白名单机制

## 快速开始

### 1. 引入依赖

```xml
<dependency>
    <groupId>com.kewen.framework.auth</groupId>
    <artifactId>auth-spring-boot-starter</artifactId>
</dependency>
<dependency>
    <groupId>com.kewen.framework.auth</groupId>
    <artifactId>auth-rbac-spring-boot-starter</artifactId>
</dependency>
```

### 2. 初始化数据库

执行 `docs/sql/auth.sql` 脚本

### 3. 配置数据源

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/your_db
    username: root
    password: password
```

### 4. 使用权限注解

```java
@RestController
@RequestMapping("/api/meeting")
public class MeetingController {
    
    // 菜单权限控制
    @AuthMenu(name = "查询会议列表")
    @GetMapping("/list")
    public Result list() {
        return Result.success(meetingService.list());
    }
    
    // 数据范围权限控制
    @AuthMenu(name = "查询我的会议")
    @AuthDataRange(businessFunction = "meeting")
    @GetMapping("/my")
    public Result myMeetings() {
        return Result.success(meetingService.list());
    }
    
    // 数据操作权限控制
    @AuthMenu(name = "编辑会议")
    @AuthDataOperation(businessFunction = "meeting", operate = "edit")
    @PostMapping("/edit")
    public Result edit(@RequestBody Meeting meeting) {
        return Result.success(meetingService.updateById(meeting));
    }
}
```

## 权限类型详解

### 菜单权限

控制用户能否访问特定的 API 接口。

**特点：**
- 启动时自动扫描并入库
- 粗粒度控制
- 可见/不可见效果

**使用方式：**
```java
@AuthMenu(name = "功能描述")
@GetMapping("/path")
public Result method() { }
```

### 数据范围权限

控制用户在查询时能看到哪些数据。

**特点：**
- 自动在 SQL 中添加权限过滤条件
- 支持 WHERE、IN、JOIN、WITH AS 等查询方式
- 细粒度数据控制

**使用方式：**
```java
@AuthDataRange(businessFunction = "业务功能")
@GetMapping("/list")
public Result list() { }
```

**生成的 SQL 示例：**
```sql
SELECT * FROM meeting_room 
WHERE id IN (
    SELECT data_id FROM sys_auth_data 
    WHERE business_function = 'meeting_room' 
    AND operate = 'unified'
    AND authority IN ('ROLE_1', 'USER_admin')
)
```

### 数据操作权限

控制用户能否对特定数据执行特定操作。

**特点：**
- 防止越权操作
- 支持同一数据的不同操作权限
- 配合数据范围权限使用

**使用方式：**
```java
@AuthDataOperation(businessFunction = "meeting_room", operate = "edit")
@PostMapping("/edit")
public Result edit(@RequestBody Meeting meeting) { }
```

### 数据权限编辑

用于编辑某条数据的权限配置。

**使用方式：**
```java
@AuthDataAuthEdit(businessFunction = "meeting_room", operate = "edit")
@PostMapping("/auth")
public Result editAuth(@RequestBody EditAuthReq req) { }
```

## 核心类说明

### AnnotationAuthHandler

权限注解的核心处理器，负责所有权限验证逻辑。

```java
public interface AnnotationAuthHandler<ID> {
    // 菜单权限验证
    boolean hasMenuAccessAuth(Collection<BaseAuth> auths, String path);
    
    // 获取数据权限表配置
    AuthDataTable getAuthDataTable();
    
    // 数据操作权限验证
    boolean hasDataOperateAuths(Collection<BaseAuth> auths, 
                                 String businessFunction, 
                                 String operate, 
                                 ID dataId);
    
    // 编辑数据权限
    void editDataAuths(ID dataId, String businessFunction, 
                       String operate, Collection<BaseAuth> auths);
}
```

### AuthDataAdaptor

数据权限适配器，提供更灵活的编程式权限操作。

```java
@Autowired
AuthDataAdaptor<Long> authDataAdaptor;

// 编辑数据权限
authDataAdaptor.editDataAuths("meeting_room", meetingId, "edit", authObject);

// 查询数据权限
IAuthObject authObject = authDataAdaptor.fillDataAuths("meeting_room", meetingId, "edit", new SimpleAuthObject());
```

## 权限配置

### 登录配置

```yaml
kewen:
  security:
    login:
      login-url: /login
      current-user-url: /currentUser
      username-parameter: username
      password-parameter: password
      token-parameter: Authorization
    session:
      maximum-sessions: 10
      max-sessions-prevents-login: true
```

### 权限数据表配置

```yaml
kewen:
  auth:
    auth-data-table:
      table-name: sys_auth_data
      id-column: id
      data-id-column: data_id
      authority-column: authority
      business-function-column: business_function
      operate-column: operate
    cache-auth: false  # 是否缓存菜单权限
```

## 权限管理 API

### 用户管理

- `GET /rabc/user/list` - 用户列表
- `GET /rabc/user/page` - 用户分页
- `POST /rabc/user/add` - 添加用户
- `POST /rabc/user/update` - 更新用户
- `POST /rabc/user/delete` - 删除用户

### 角色管理

- `GET /rabc/role/list` - 角色列表
- `GET /rabc/role/page` - 角色分页
- `POST /rabc/role/add` - 添加角色
- `POST /rabc/role/update` - 更新角色
- `POST /rabc/role/delete` - 删除角色

### 部门管理

- `GET /rabc/dept/list` - 部门列表
- `GET /rabc/dept/page` - 部门分页
- `POST /rabc/dept/add` - 添加部门
- `POST /rabc/dept/update` - 更新部门
- `POST /rabc/dept/delete` - 删除部门

### 菜单管理

- `GET /menu/route/tree` - 获取路由菜单树
- `GET /menu/api/tree` - 获取 API 菜单树
- `POST /menu/api/update` - 更新菜单

### 当前用户

- `GET /currentUser` - 获取当前用户信息
- `GET /currentUser/routeMenus` - 获取当前用户菜单
- `POST /currentUser/updatePassword` - 修改密码

## 扩展开发

### 自定义权限处理器

实现 `AnnotationAuthHandler` 接口：

```java
@Component
public class CustomAnnotationAuthHandler implements AnnotationAuthHandler<Long> {
    @Override
    public boolean hasMenuAccessAuth(Collection<BaseAuth> auths, String path) {
        // 自定义菜单权限逻辑
    }
    
    // 实现其他方法...
}
```

### 扩展权限维度

添加新的权限维度（如岗位）：

1. 创建权限实体类
2. 扩展权限集合对象
3. 重写用户加载逻辑
4. 更新权限编辑入参

详见：[权限扩展开发](./权限扩展开发.md)

## 最佳实践

1. **菜单权限用于高级接口控制**：如增删改操作
2. **数据权限用于业务数据控制**：如查询范围、编辑权限
3. **合理划分业务功能**：businessFunction 应清晰明确
4. **注意事务管理**：权限编辑时需保证事务一致性
5. **避免权限混淆**：菜单权限和数据权限各司其职
