# 开发最佳实践

## 项目结构规范

### 包结构

```
com.kewen.framework.yourmodule
├── controller/           # 控制器层
├── service/             # 服务层
│   └── impl/            # 服务实现
├── mapper/              # 数据访问层
├── entity/              # 实体类
├── model/               # 数据模型
│   ├── req/             # 请求对象
│   └── resp/            # 响应对象
├── config/              # 配置类
└── utils/               # 工具类
```

### 命名规范

#### 类命名

- **Controller**: `XxxController`
- **Service**: `XxxService` (接口), `XxxServiceImpl` (实现)
- **Mapper**: `XxxMapper`
- **Entity**: `Xxx` (与表名对应)
- **DTO/VO**: `XxxReq`, `XxxResp`, `XxxDTO`

#### 方法命名

- **查询单个**: `getById`, `getByXxx`
- **查询列表**: `list`, `listByXxx`
- **分页查询**: `page`, `pageByXxx`
- **新增**: `save`, `add`, `create`
- **更新**: `update`, `modify`
- **删除**: `delete`, `remove`
- **统计**: `count`

### 权限注解使用

```java
@RestController
@RequestMapping("/api/meeting")
@AuthMenu(name = "会议管理")
public class MeetingController {
    
    // 菜单权限
    @AuthMenu(name = "查询列表")
    @GetMapping("/list")
    public Result list() {
        return Result.success(meetingService.list());
    }
    
    // 数据范围权限
    @AuthMenu(name = "查询我的会议")
    @AuthDataRange(businessFunction = "meeting")
    @GetMapping("/my")
    public Result myMeetings() {
        return Result.success(meetingService.list());
    }
    
    // 数据操作权限
    @AuthMenu(name = "编辑会议")
    @AuthDataOperation(businessFunction = "meeting", operate = "edit")
    @PostMapping("/edit")
    public Result edit(@RequestBody Meeting meeting) {
        return Result.success(meetingService.updateById(meeting));
    }
    
    // 权限编辑
    @AuthMenu(name = "设置会议权限")
    @AuthDataAuthEdit(businessFunction = "meeting", operate = "edit")
    @PostMapping("/auth")
    public Result editAuth(@RequestBody EditAuthReq req) {
        return Result.success();
    }
}
```

## Service 层规范

### 接口定义

```java
public interface XxxService extends IService<Xxx> {
    
    /**
     * 根据条件查询
     */
    List<Xxx> listByCondition(XxxQueryReq req);
    
    /**
     * 分页查询
     */
    PageResult<Xxx> page(XxxPageReq req);
}
```

### 实现类

```java
@Service
public class XxxServiceImpl extends ServiceImpl<XxxMapper, Xxx> 
        implements XxxService {
    
    @Override
    public List<Xxx> listByCondition(XxxQueryReq req) {
        LambdaQueryWrapper<Xxx> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(StringUtils.isNotBlank(req.getName()), Xxx::getName, req.getName())
               .eq(req.getStatus() != null, Xxx::getStatus, req.getStatus())
               .orderByDesc(Xxx::getCreateTime);
        return list(wrapper);
    }
}
```

### 使用 AuthDataAdaptor

```java
@Service
public class MeetingServiceImpl extends ServiceImpl<MeetingMapper, Meeting> 
        implements MeetingService {
    
    @Autowired
    private AuthDataAdaptor<Long> authDataAdaptor;
    
    /**
     * 创建会议并设置权限
     */
    @Transactional(rollbackFor = Exception.class)
    public Meeting createMeeting(Meeting meeting, IAuthObject authObject) {
        // 保存会议
        save(meeting);
        
        // 编辑数据权限
        authDataAdaptor.editDataAuths(
            "meeting",           // 业务功能
            meeting.getId(),     // 数据ID
            "edit",              // 操作类型
            authObject           // 权限对象
        );
        
        return meeting;
    }
    
    /**
     * 查询会议权限
     */
    public IAuthObject getMeetingAuth(Long meetingId) {
        SimpleAuthObject authObject = new SimpleAuthObject();
        return authDataAdaptor.fillDataAuths(
            "meeting",
            meetingId,
            "edit",
            authObject
        );
    }
}
```

## 异常处理

### 自定义异常

```java
public class BusinessException extends RuntimeException {
    
    private int code;
    
    public BusinessException(String message) {
        super(message);
        this.code = 500;
    }
    
    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }
}
```

### 全局异常处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    private static final Logger log = LoggerFactory.getLogger(GlobalExceptionHandler.class);
    
    @ExceptionHandler(BusinessException.class)
    public Result<?> handleBusinessException(BusinessException e) {
        log.error("业务异常: {}", e.getMessage());
        return Result.failed(e.getCode(), e.getMessage());
    }
    
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<?> handleValidException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
                .map(FieldError::getDefaultMessage)
                .findFirst()
                .orElse("参数错误");
        return Result.failed(400, message);
    }
    
    @ExceptionHandler(Exception.class)
    public Result<?> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.failed(500, "系统异常");
    }
}
```

## 事务管理

### 声明式事务

```java
@Service
public class XxxServiceImpl implements XxxService {
    
    @Transactional(rollbackFor = Exception.class)
    public void createWithRelation(Xxx xxx, List<Relation> relations) {
        // 保存主表
        save(xxx);
        
        // 保存关联表
        relationService.saveBatch(relations);
        
        // 编辑权限
        authDataAdaptor.editDataAuths("xxx", xxx.getId(), "edit", authObject);
    }
}
```

### 注意事项

- 事务方法必须是 public
- 同类方法调用事务不生效（需要注入自身或拆分到不同类）
- 捕获异常会导致事务不回滚（需手动回滚或抛出异常）

## 日志规范

### 日志级别

- **ERROR**: 错误信息，需要立即处理
- **WARN**: 警告信息，可能需要关注
- **INFO**: 重要业务流程
- **DEBUG**: 调试信息

## 性能优化

### 查询优化

```java
// 避免 SELECT *
LambdaQueryWrapper<Xxx> wrapper = new LambdaQueryWrapper<>();
wrapper.select(Xxx::getId, Xxx::getName);

// 合理使用索引
wrapper.eq(Xxx::getStatus, 1);  // status 有索引

// 分页查询
Page<Xxx> page = new Page<>(1, 10);
IPage<Xxx> result = page(page, wrapper);
```

        return getById(id);

## 安全建议

1. **参数校验**: 使用 `@Validated` 和校验注解
2. **SQL 注入**: 使用参数化查询，避免拼接 SQL
3. **XSS 防护**: 输入输出过滤
4. **权限控制**: 合理使用权限注解
5. **敏感信息**: 密码加密存储，日志脱敏
