# API 接口概览

## 接口说明

Kewen Framework 提供了完整的 RESTful API 接口，主要包括权限管理、用户管理、文件存储等功能。

## 基础信息

- **基础路径**：根据具体部署配置
- **认证方式**：Session/Token
- **数据格式**：JSON
- **字符编码**：UTF-8

## 统一返回格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 权限管理接口

### 用户管理 `/rabc/user`

#### 获取用户列表

```
GET /rabc/user/list
```

**响应：**
```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "username": "admin",
      "realName": "管理员",
      "status": 1
    }
  ]
}
```

#### 用户分页查询

```
GET /rabc/user/page?current=1&size=10
```

**参数：**
- `current`: 当前页码
- `size`: 每页大小

#### 添加用户

```
POST /rabc/user/add
```

**请求体：**
```json
{
  "username": "newuser",
  "password": "123456",
  "realName": "新用户",
  "deptId": 1,
  "status": 1
}
```

#### 更新用户

```
POST /rabc/user/update
```

#### 删除用户

```
POST /rabc/user/delete
```

**请求体：**
```json
{
  "id": 1
}
```

## 认证接口

### 登录

```
POST /login
```

**请求体：**
```json
{
  "username": "admin",
  "password": "123456",
  "remember-me": true
}
```

**响应：**
```json
{
  "code": 200,
  "data": {
    "token": "xxx-xxx-xxx",
    "user": {
      "id": 1,
      "username": "admin"
    }
  }
}
```

### 退出登录

```
POST /logout
```

## 文件存储接口

### 上传文件

```
POST /storage/upload
```

**请求：**
- Content-Type: `multipart/form-data`
- 参数：`file` (文件)

**响应：**
```json
{
  "code": 200,
  "data": {
    "id": "file001",
    "url": "https://xxx.qiniucdn.com/file001.jpg",
    "filename": "test.jpg",
    "size": 102400
  }
}
```

### 下载文件

```
GET /storage/download/{fileId}
```

### 删除文件

```
POST /storage/delete
```

**请求体：**
```json
{
  "fileId": "file001"
}
```

## 分页请求格式

```json
{
  "current": 1,
  "size": 10,
  "sortField": "createTime",
  "sortOrder": "desc"
}
```

```json
{
  "code": 200,
  "data": {
    "records": [],
    "total": 100,
    "current": 1,
    "size": 10,
    "pages": 10
  }
}
```

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 401 | 未认证 |
| 403 | 无权限 |
| 500 | 服务器错误 |

## 认证说明

### Token 认证

在请求头中添加 Token：

```
Authorization: Bearer xxx-xxx-xxx
```

### Session 认证

基于 Cookie-Session 机制，登录成功后自动维护 Session。

## 白名单接口

以下接口无需认证即可访问：

- `/login` - 登录
- 静态资源
- 配置文件中定义的白名单接口

## 接口权限

所有接口（除白名单外）都需要认证，部分接口需要特定权限：

- 标注 `@AuthMenu` 的接口需要菜单权限
- 标注 `@AuthDataRange` 的接口有数据范围限制
- 标注 `@AuthDataOperation` 的接口需要数据操作权限
