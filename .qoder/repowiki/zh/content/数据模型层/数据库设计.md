# 数据库设计

## 数据库概览

Kewen Framework 使用 MySQL 作为主要数据库，包含权限管理、文件存储、请求日志等核心表。

## 权限系统表

### 用户表 (sys_user)

```sql
CREATE TABLE `sys_user` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` varchar(50) NOT NULL COMMENT '用户名',
  `password` varchar(100) NOT NULL COMMENT '密码',
  `real_name` varchar(50) DEFAULT NULL COMMENT '真实姓名',
  `dept_id` bigint(20) DEFAULT NULL COMMENT '部门ID',
  `email` varchar(100) DEFAULT NULL COMMENT '邮箱',
  `phone` varchar(20) DEFAULT NULL COMMENT '手机号',
  `status` tinyint(4) DEFAULT '1' COMMENT '状态 0-禁用 1-启用',
  `deleted` tinyint(4) DEFAULT '0' COMMENT '逻辑删除 0-未删除 1-已删除',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 用户凭证表 (sys_user_credential)

```sql
CREATE TABLE `sys_user_credential` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT '凭证ID',
  `user_id` bigint(20) NOT NULL COMMENT '用户ID',
  `credential_type` varchar(20) NOT NULL COMMENT '凭证类型 PASSWORD/SMS/SSO',
  `credential_value` varchar(200) NOT NULL COMMENT '凭证值',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户凭证表';
```

### 角色表 (sys_role)

## Spring Session 表

### Session 表 (SPRING_SESSION)

```sql
CREATE TABLE `SPRING_SESSION` (
  `PRIMARY_ID` char(36) NOT NULL,
  `SESSION_ID` char(36) NOT NULL,
  `CREATION_TIME` bigint(20) NOT NULL,
  `LAST_ACCESS_TIME` bigint(20) NOT NULL,
  `MAX_INACTIVE_INTERVAL` int(11) NOT NULL,
  `EXPIRY_TIME` bigint(20) NOT NULL,
  `PRINCIPAL_NAME` varchar(100) DEFAULT NULL,
  PRIMARY KEY (`PRIMARY_ID`),
  UNIQUE KEY `SPRING_SESSION_IX1` (`SESSION_ID`),
  KEY `SPRING_SESSION_IX2` (`EXPIRY_TIME`),
  KEY `SPRING_SESSION_IX3` (`PRINCIPAL_NAME`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 ROW_FORMAT=DYNAMIC;

CREATE TABLE `SPRING_SESSION_ATTRIBUTES` (
  `SESSION_PRIMARY_ID` char(36) NOT NULL,
  `ATTRIBUTE_NAME` varchar(200) NOT NULL,
  `ATTRIBUTE_BYTES` blob NOT NULL,
  PRIMARY KEY (`SESSION_PRIMARY_ID`,`ATTRIBUTE_NAME`),
  CONSTRAINT `SPRING_SESSION_ATTRIBUTES_FK` FOREIGN KEY (`SESSION_PRIMARY_ID`) REFERENCES `SPRING_SESSION` (`PRIMARY_ID`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8 ROW_FORMAT=DYNAMIC;
```

## 表关系说明

### 权限系统关系

```
用户 (sys_user)
  ├── 部门 (sys_dept) [多对一]
  ├── 角色 (sys_role) [多对多，通过 sys_user_role]
  │     └── 菜单 (sys_menu) [多对多，通过 sys_role_menu]
  └── 数据权限 (sys_auth_data) [一对多]
```

### 核心关联

1. **用户权限获取流程**：
   - 用户 → 角色 → 菜单权限
   - 用户 → 数据权限

2. **数据权限控制**：
   - 通过 `sys_auth_data` 表控制
   - 关联业务数据 ID 和权限标识

## 索引设计

### 关键索引

1. **用户表**
   - `uk_username`: 用户名唯一索引

2. **角色表**
   - `uk_role_code`: 角色编码唯一索引

3. **数据权限表**
   - `idx_data_id`: 数据 ID 索引
   - `idx_business_function`: 业务功能索引
   - `idx_authority`: 权限标识索引

4. **请求日志表**
   - `idx_create_time`: 创建时间索引
   - `idx_user_id`: 用户 ID 索引
